Getting started
  1. Plug the glove in over USB‑C.
  2. Click Connect Device and pick the XIAO ESP32-C3's serial port from the browser prompt.
  3. Adjust the panels — the payload preview updates live as you move each slider, flip a toggle, or edit a keybind.
  4. Click Send to Device to push the values into RAM over serial.

Settings apply immediately but live in RAM only — they reset to firmware defaults on the next power cycle unless your sketch writes them to flash.

About this panel

Live in firmware vs Not in firmware yet tags reflect what readSerialConfig() in the current build actually reads. Pending panels are dimmed but still fully interactive — sending them is harmless (unrecognized JSON keys are just skipped), so they're ready to go live the moment the matching firmware code exists.

The Cursor and Clutch & Zoom panels use the field names, ranges, and defaults exactly as given — no assumed units or scales there. maxCursorSpeed is shown as a bare number since no unit was specified; relabel it if your firmware treats it as, say, px/frame or deg/s.

The Haptics panel's hapDur field extends the original brief's four-key example, since a duration slider was called for but not spelled out in the sample JSON.

Keybind slots default to the four core CAD gestures from the original brief (Extrude / Cut / Revolve / Normal-To) plus one open slot — all plain text fields, ready for your real bindings whenever you send them over.

Baud rate defaults to 115200, the typical ESP32 Serial.begin() rate — matches the current firmware's Serial.begin(115200) exactly.
